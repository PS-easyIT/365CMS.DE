Exchange Online | EWS Abschaltung Mai 2027 – Jetzt handeln!

Das Ende von Exchange Web Services ist kein theoretisches Szenario mehr. Microsoft hat den endgültigen Zeitplan veröffentlicht – und wer jetzt nicht mit der Analyse beginnt, riskiert Produktionsausfälle ab Dezember 2026.

AH
Andreas Hepp
Senior IT Administrator · Microsoft MVP
· 7. Februar 2026 · 8 Min. Lesezeit

Was ist EWS und warum wird es abgeschaltet?

Exchange Web Services (EWS) ist eine SOAP-basierte API, die Microsoft mit Exchange Server 2007 einführte. Über fast zwei Jahrzehnte war sie die primäre Schnittstelle für Drittanbieter-Anwendungen – E-Mail-Clients, Backup-Lösungen, CRM-Systeme und unzählige Unternehmens-Applikationen.

⚠️
Kritischer Termin: 1. Mai 2027 Microsoft schaltet EWS vollständig ab. Alle Anfragen werden danach mit HTTP 410 Gone beantwortet – ohne Ausnahme oder Verlängerung.

Der offizielle Zeitplan

Microsoft hat den Abschaltungsplan in mehreren Message Center-Einträgen (MC681916, MC775941) schrittweise präzisiert:

DatumMaßnahmeStatus
Oktober 2023Neue EWS-Verbindungen für neue Tenants blockiertAbgeschlossen
Januar 2025EWS-Throttling für bestehende Tenants verschärftAbgeschlossen
Dez. 2026EWS wird read-onlyAusstehend
1. Mai 2027Vollständige AbschaltungAusstehend

„Wer wartet, verliert. EWS-abhängige Systeme zu inventarisieren dauert in komplexen Umgebungen Wochen – nicht Stunden."

Andreas Hepp · ITPortal.DE

Betroffene Systeme per PowerShell analysieren

Bevor die Migration beginnt, muss ein vollständiges Inventar aller EWS-nutzenden Systeme erstellt werden. Das folgende Skript extrahiert alle EWS-Verbindungen der letzten 7 Tage aus dem Unified Audit Log:

PowerShell 
# EWS-Verbindungen der letzten 7 Tage analysieren
Connect-ExchangeOnline -ShowBanner:$false

$startDate = (Get-Date).AddDays(-7)
$endDate   = Get-Date

$ewsLogs = Search-UnifiedAuditLog `
    -StartDate $startDate `
    -EndDate   $endDate `
    -Operations "MailItemsAccessed" `
    -ResultSize 5000

# Filtern und gruppieren
$ewsLogs.AuditData |
    ConvertFrom-Json |
    Where-Object { $_.ClientInfoString -match "EWS" } |
    Group-Object -Property ClientInfoString |
    Sort-Object Count -Descending |
    Select-Object Name, Count |
    Format-Table -AutoSize
ℹ️
Voraussetzung: Audit-Logging aktiv Das Unified Audit Log muss aktiviert sein. Prüfe mit Get-AdminAuditLogConfig | Select UnifiedAuditLogIngestionEnabled.

Migration zu Microsoft Graph API

Der empfohlene Migrationspfad führt zur Microsoft Graph API – REST-basiert, OAuth2-gesichert und aktiv weiterentwickelt. Die gute Nachricht: Für die meisten EWS-Funktionen gibt es eine direkte Graph-Entsprechung.

✅
Empfohlene Sofortmaßnahmen EWS-Inventar erstellen → Hersteller-Roadmaps prüfen → Graph-Migrationsplan pro App → Test-Tenant einrichten → Intern kommunizieren.
